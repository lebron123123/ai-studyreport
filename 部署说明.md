## 必做：D1 数据库新建一张日志表

Cloudflare → D1 → 你的库 → Console，执行：

```sql
CREATE TABLE IF NOT EXISTS rag_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER,
  query TEXT,
  category TEXT,
  hit_titles TEXT,
  hit_count INTEGER,
  top_score REAL,
  created_at INTEGER NOT NULL
);
```

不建这张表：检索仍能正常用（记录日志的动作会静默失败，不影响检索本身），但看板和日志页会是空的。所以建议建上。

## 新增功能

进后台 → RAG知识库 → 检索测试那一行，多了两个入口：

**📊 知识库看板**：一屏看清家底——累计检索次数、近7天检索次数；各分类的文件数/向量块数/停用数；各等级文件数；🔥最常被检索命中的文件；❄从未被命中的文件（帮你揪出冗余/低质文档）。

**📜 检索日志**：最近200条检索记录——什么时间、搜了什么词、限定了哪个分类、命中几条、最高分、命中了哪些文件。这是国企最看重的"可追溯"。

**按分类检索**：检索测试框旁多了分类下拉，可只在"政策文件"或"制度规范"里检索。

## 给领导汇报的话术

> 第二阶段完成知识库的可观测与可追溯能力：新增检索日志留痕（每次检索的时间、检索词、命中文件全部记录可查）、知识库总览看板（分类分级统计、热门与冷门文件识别）、按分类精准检索。至此知识库具备了合规审计所需的日志基础，管理员可随时追溯任一次问答的来源依据。
