# 知识库第一阶段（文档治理）部署说明 

## 改动的文件（2个，注意重构后的目录）

| 文件 | 传到哪里 | 操作 |
|---|---|---|
| `admin.html` | 仓库**根目录**（覆盖原来的 admin.html） | 替换 |
| `rag.js` | 仓库的 **functions/api/** 目录下（覆盖原来的 functions/api/rag.js） | 替换 |

⚠️ 注意：rag.js 有两个位置——**根目录也有一个 rag.js（前端图表用），不要动它**；要替换的是 **functions/api/rag.js**（后端接口）。别传错。

## 关键：数据库要新建一张表（必须做，否则新功能报错）

登录 Cloudflare → D1 数据库 → 你的数据库 → Console，执行这条 SQL：

```sql
CREATE TABLE IF NOT EXISTS rag_files_v2 (
  title TEXT PRIMARY KEY,
  ids TEXT NOT NULL,
  chunks INTEGER NOT NULL,
  category TEXT DEFAULT '未分类',
  level INTEGER DEFAULT 2,
  enabled INTEGER DEFAULT 1,
  created_at INTEGER NOT NULL
);
```

## 关于你已经入库的那些文件

新系统用的是新表 rag_files_v2，所以**你之前入库的文件不会出现在新的文件列表里**（它们还在向量库里、检索仍能命中，只是没有分类/等级/启停管理）。两个选择：
- **不管**：老文件继续能被检索到（按默认参考等级处理），新传的文件才有完整治理功能
- **想全部纳入管理**：重建一次 Vectorize 索引（命令行 delete 再 create），然后按分类重新上传——干净但费时

建议：如果老文件不多，重传一次最省心。

## 新增功能一览

1. **上传时选分类+权威等级**：分类（可研报告/政策文件/制度规范/业务逻辑/会议纪要/其他）、等级（权威/参考/存档）
2. **文件列表**：显示每个文件的分类标签、等级、块数；可**按分类筛选**；每个文件可**停用/启用**（停用=保留存档但不参与检索，不用删除）
3. **检索加权**：权威等级文件检索得分×1.15、存档×0.85——政策文件自动优先于历史资料
4. **检索溯源**：检索测试结果显示每条来自哪个文件、什么分类、什么等级
