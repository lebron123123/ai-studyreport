# 部署说明：可研报告生成工具（免费托管版）  

本项目分两部分：
- `public/index.html` —— 网站前端（用户看到的页面）
- `functions/api/generate.js` —— 后端函数（保管API Key、做访问口令校验和每日限额，再转发给Anthropic）

用 **Cloudflare Pages** 一起托管，前端和这个小后端函数都在免费额度内。

---

## 第一步：申请 Anthropic API Key（这部分不是免费的）

1. 打开 console.anthropic.com，注册账号
2. 进入 Billing，充值一点额度（这是唯一真正花钱的地方，按实际生成的字数计费）
3. 在 API Keys 页面创建一个 Key，先复制保存好（离开页面后无法再次查看完整Key）

> 建议先只充值一个小额度（比如够测试用的），观察几天实际消耗，再决定要不要加大。

## 第二步：注册 Cloudflare，创建 Pages 项目

1. 打开 dash.cloudflare.com，免费注册
2. 左侧菜单找到 **Workers & Pages** → **创建应用程序** → **Pages** → **直接上传**
3. 把本项目文件夹（包含 `public` 和 `functions` 两个目录）整体上传
   - 如果你熟悉 Git，也可以把项目推到 GitHub，用"连接到Git仓库"的方式，之后每次改代码推送就会自动重新部署，更方便迭代
4. 部署完成后，Cloudflare 会给你一个免费的 `xxx.pages.dev` 域名，这就是网站地址，可以直接发给同事

## 第三步：创建一个 KV 存储（用来记录每日调用次数）

1. 左侧菜单 **Workers & Pages** → **KV** → 创建一个命名空间，随便取名，比如 `USAGE_KV`
2. 回到你的 Pages 项目 → **设置** → **函数** → **KV 命名空间绑定**
3. 变量名填 `USAGE_KV`，绑定刚才创建的命名空间

## 第四步：配置环境变量（这一步最关键，Key就是存在这里）

在 Pages 项目 → **设置** → **环境变量** 里添加：

| 变量名 | 值 | 说明 |
|---|---|---|
| `ANTHROPIC_API_KEY` | 你第一步申请的Key | 一定要加密存储（Cloudflare里选择"加密"） |
| `ACCESS_CODE` | 你自己定的一个口令，比如 `fkyj2026` | 只把这个口令告诉需要用的同事 |

保存后重新部署一次，让新的环境变量生效。

## 第五步：打开网站试用

访问 `xxx.pages.dev`，输入你设置的访问口令，就能开始走完整的四步生成流程了。

---

## 关于"控制成本"，还可以做但目前先没做的事

- 目前的每日限额（`functions/api/generate.js` 里的 `DAILY_LIMIT`）是全站共享的一个数字，不是按人算的，先用着看效果，之后可以改成按口令分别计数
- 访问口令目前是"错了就在生成时报错"，还没有做成一进网站就校验对错的形式，如果同事反馈体验不好，可以再补一个校验接口
- 如果之后要换成真正的用户登录（每个人一个账号），或者要把某几章的数字接上真实财务测算而不是AI示例框架，这些都是在现在这个骨架上继续加，不用推倒重来


# 升级版（v3）更新说明

本次升级的核心：新增两个领域、章节结构按真实报告细化、支持表格。

## ⚠ 最重要：这次多了一个新文件 outlines.js

相比你现在线上的版本，这一版**多了一个文件 `outlines.js`**（放在和 index.html 同一层，也就是仓库根目录）。这个文件装的是两个领域的章节大纲。**如果只更新 index.html 而漏传 outlines.js，网站会打不开领域选择页**，务必一起上传。

## 需要更新/新增的文件清单

| 文件 | 在仓库里的位置 | 操作 |
|---|---|---|
| `index.html` | 根目录 | 替换（整个换掉） |
| `outlines.js` | 根目录 | **新增**（仓库里原来没有，需要新建） |
| `functions/api/generate.js` | functions/api/ | 保持不变（和上一版一样，无需动） |
| `functions/api/verify.js` | functions/api/ | 保持不变 |

### 在 GitHub 上怎么新增 outlines.js
1. 打开仓库根目录 → Add file → Create new file
2. 文件名填 `outlines.js`
3. 把本文件夹里的 `outlines.js` 内容整个粘贴进去
4. Commit

然后再把 `index.html` 的内容替换掉，一起提交。Cloudflare 会自动重新部署。

## 环境变量和 KV

**都不用动**，和现在完全一样（`ACCESS_CODE`、`DEEPSEEK_API_KEY`、`USAGE_KV` 绑定继续沿用）。

## 这一版新增/改进了什么

1. **多了第一步"选择领域"**：进入后先选"保障房·非居改保"或"保障房·新建"，两个领域用各自的专属章节大纲。
2. **章节结构大幅细化**：从旧版"一章一段"改成"一章下含多个子标题，逐个子标题撰写"。非居改保有 13 章约 40 个子标题，新建有 13 章约 40 个子标题，单份报告篇幅和结构颗粒度显著提升。
3. **数据类子章节自动生成表格**：投资估算、财务评价等涉及数字的子标题，会生成结构完整的表格框架，数字统一填"待填"，并醒目标注需替换为真实测算——既贴合真实报告的表格形态，又守住"不编造数字"的边界。
4. **Word 导出支持表格和多级标题**：导出的 .doc 里，一级章节、子标题、表格都会保留层级和边框。 

## 关于篇幅

单份报告子标题多、每节 500-800 字，生成时会**逐个子标题调用 AI**，所以一份完整报告大约要调用 40 次左右、耗时几分钟，也会相应消耗更多 DeepSeek 额度。如果只想快速看某几章，可以在"章节范围"那一步取消勾选不需要的整章。

## 关于"待填"数字（一定要跟领导讲清楚的边界）

这一版能把报告的**结构、行文、表格形态**做得非常接近真实报告，但表格里的具体数字仍然是"待填"占位——因为投资估算、经营收入、回报率这类数字必须来自真实测算，AI 编出来的数字看着专业却经不起审查。所以这套工具的定位是"把 80% 的结构和文字骨架自动搭好，剩下的真实数据由专业人员填入并复核签发"，而不是"一键出终版报告"。这条边界守住了，批量生产才有意义。
# 升级版（v4）更新说明：接入真实财务测算引擎

## 这一版做了什么

把你 Streamlit 测算器（calculator3.py）里"非居改保"的完整测算逻辑，**1:1 翻译成了网页里的 JavaScript 测算引擎**，并已做交叉验证：同一组参数下，网页算出的总收入、总成本、税金、净利润、净现值、IRR 与 Python 原版**逐项完全一致**（精确到小数点后四位的中间值都一致）。

流程从 5 步变成 6 步，新增第 3 步"财务测算"：

1. 选择领域 → 2. 项目信息 → **3. 财务测算（新增）** → 4. 章节范围 → 5. 逐章生成 → 6. 复核签发

## 财务测算步骤怎么用

- 选择"保障房·非居改保"领域时，第 3 步会出现完整的测算参数表单（住宅面积、起始租金、递增规则、出租率爬坡、收楼单价、装修造价与间隔、套数、运营成本、借款/计息/利率折扣、折现率、还款计划等，默认值与你 Streamlit 版一致）
- 点"执行测算"→ 当场算出全周期收入/成本/税金/损益/现金流/IRR/NPV/回正年份，绿色汇总条展示结果
- **生成报告时，"投资估算""财务评价"等数据类章节会自动拿到真实测算结果**——AI 严格引用这些算出来的数字写分析、出表格，不再是"待填"
- 不想测算也可以直接"跳过"，财务章节回到原来的"待填"框架模式
- 导出 Word 时，报告末尾会自动附上"财务测算明细"附表（分年收入/成本/税金/净利润/现金流全表）

## 需要更新的文件

| 文件 | 位置 | 操作 |
|---|---|---|
| `index.html` | 根目录 | 替换 |
| `nrcalc.js` | 根目录 | **新增**（测算引擎，必须上传，否则测算步骤会报错） |
| `outlines.js` | 根目录 | 不变（如果 v3 已上传） |
| `functions/api/*` | functions/api/ | 不变 |

环境变量、KV 全部不用动。

## 测算口径说明（给使用者/领导看）

网页测算引擎与内部 Streamlit 测算器采用同一套公式：
- 收入：住宅面积×租金单价×出租率×月数，租金按跨度递增，出租率爬坡后进入稳定期，收入按 9% 增值税价税分离
- 成本四项：收楼成本（面积×收楼单价×出租率）、工程费用（首装+按间隔重装，按运营期总月数摊销）、运营费用（单套成本×套数，支持递增，首年含开办费）、财务费用（按计息本金比例缩放的年均余额计息，含利率折扣）
- 税金：销项-进项的增值税、12% 附加、首年印花税
- 损益：含五年弥补亏损规则的所得税（25%）与净利润
- 现金流：年中折现净现值、牛顿迭代求解 IRR（与 Excel 口径一致）

## 边界的变化（重要）

之前的边界是"AI 不碰数字，全部待填"。这一版升级为：**数字由确定性代码计算，AI 只负责引用和解读**——这是更强的形态：数字可复算、可追溯（口径就是上面这套公式），AI 不再有编数字的空间。但仍要提醒使用者：测算结果的可靠性取决于输入参数的真实性，参数本身（租金水平、出租率假设、造价）仍需专业判断和依据，签发前的人工复核环节不能省。

## 后续可以继续做的

- "保障房·新建"领域的测算模块（对应你 Streamlit 里的出租类/出售类/租售结合类逻辑），做法与本次相同
- 敏感性分析（租金±10%、出租率±5% 对 IRR/NPV 的影响表）
- 出租率爬坡支持多年逐年设置（目前网页版简化为首年一档+稳定期，Streamlit 版支持多年爬坡）

# 升级版（v5）更新说明：index.html 内联脚本按功能拆成独立文件

## 这一版做了什么

`index.html` 原来把全部 JS 逻辑（路由、测算、报告生成、审核、POI、图表、导出、账号体系……）塞在一个 2600 多行的内联 `<script>` 里。这一版把它按功能拆成 9 个独立 `.js` 文件，`index.html` 现在只剩页面结构和一串 `<script src="...">` 引用，内联脚本只留一行启动调用。

**部署方式完全没变**——这些新文件和 `nrcalc.js`/`rentcalc.js`/`salecalc.js`/`outlines.js` 一样，都是普通脚本文件，跟着 `index.html` 一起原样上传/推送即可，不需要 `npm install`、不需要构建、没有打包步骤。`functions/api/*`、环境变量、KV 绑定全部不用动。

## 需要一并上传的新文件

| 文件 | 内容 |
|---|---|
| `index.html` | 替换（已改为通过 `<script src>` 引用下面这些文件） |
| `calc.js` | **新增**：测算表单渲染/读取、明细表规格、灵敏度分析、AI问答 |
| `report.js` | **新增**：领域/项目信息步骤、章节生成、知识库检索、草稿存档 |
| `review.js` | **新增**：智能审查上传解析、硬规则检查、AI深度评审 |
| `poi.js` | **新增**：周边配套、竞品调研、AI选址建议 |
| `charts.js` | **新增**：现金流图表、竞品价格对比条形图 |
| `export.js` | **新增**：测算Excel导出、Word导出（含图表转PNG） |
| `metrics.js` | **新增**：测算评分规则、自定义指标公式求值 |
| `app.js` | **新增**：步骤路由、TOC导航、首页、全局事件绑定 |
| `auth.js` | **新增**：登录态、云端自动保存、项目管理、启动引导 |

九个文件缺一个都会导致对应功能报错（比如漏传 `auth.js` 会连登录都进不去），和当年补传 `outlines.js` 是同一类注意事项。

## 关于 package.json / tests/ 目录

同一次改动里还加了 `package.json` 和 `tests/calc-engines.test.js`——这是给三个测算引擎（`nrcalc.js`/`rentcalc.js`/`salecalc.js`）用的本地回归测试（`npm test`），跑在你自己电脑上，验证重构没有改变任何测算数字。**这两样纯粹是开发时用的，和线上部署无关**：Cloudflare Pages 不会因为仓库里多了 package.html 就去跑构建，直接上传或 Git 推送的行为跟以前一模一样，不上传这两样也完全不影响网站运行。

## 拆分原则（后续继续拆分/加功能时可以参照）

- 只拆文件边界，不改函数实现——所有文件仍是普通脚本（不是 ES module），浏览器里多个 `<script>` 标签本来就共享同一个全局作用域，函数之间可以互相调用，不需要 `import`/`export`，也不需要打包工具。
- 新功能优先放进已有的对应模块文件（测算相关进 `calc.js`，导出相关进 `export.js`……），实在归不进现有模块再新开一个 `.js` 文件、在 `index.html` 里加一行 `<script src>` 引用即可，不涉及任何构建配置改动。
