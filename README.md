# 部署说明：可研报告生成工具（免费托管版）

本项目分两部分：
- `public/index.html` —— 网站前端（用户看到的页面）
- `functions/api/generate.js` —— 后端函数（保管API Key、做访问口令校验和每日限额，再转发给Anthropic）

用 **Cloudflare Pages** 一起托管，前端和这个小后端函数都在免费额度内。

---

## 第一步：申请 Anthropic API Key（这部分不是免费的）

1. 打开 console.anthropic.com，注册账号
2. 进入 Billing，充值一点额度（这是唯一真正花钱的地方，按实际生成的字数计费）
3. 在 API Keys 页面创建一个 Key，先复制保存好（离开页面后无法再次查看完整Key）

> 建议先只充值一个小额度（比如够测试用的），观察几天实际消耗，再决定要不要加大。

## 第二步：注册 Cloudflare，创建 Pages 项目

1. 打开 dash.cloudflare.com，免费注册
2. 左侧菜单找到 **Workers & Pages** → **创建应用程序** → **Pages** → **直接上传**
3. 把本项目文件夹（包含 `public` 和 `functions` 两个目录）整体上传
   - 如果你熟悉 Git，也可以把项目推到 GitHub，用"连接到Git仓库"的方式，之后每次改代码推送就会自动重新部署，更方便迭代
4. 部署完成后，Cloudflare 会给你一个免费的 `xxx.pages.dev` 域名，这就是网站地址，可以直接发给同事

## 第三步：创建一个 KV 存储（用来记录每日调用次数）

1. 左侧菜单 **Workers & Pages** → **KV** → 创建一个命名空间，随便取名，比如 `USAGE_KV`
2. 回到你的 Pages 项目 → **设置** → **函数** → **KV 命名空间绑定**
3. 变量名填 `USAGE_KV`，绑定刚才创建的命名空间

## 第四步：配置环境变量（这一步最关键，Key就是存在这里）

在 Pages 项目 → **设置** → **环境变量** 里添加：

| 变量名 | 值 | 说明 |
|---|---|---|
| `ANTHROPIC_API_KEY` | 你第一步申请的Key | 一定要加密存储（Cloudflare里选择"加密"） |
| `ACCESS_CODE` | 你自己定的一个口令，比如 `fkyj2026` | 只把这个口令告诉需要用的同事 |

保存后重新部署一次，让新的环境变量生效。

## 第五步：打开网站试用

访问 `xxx.pages.dev`，输入你设置的访问口令，就能开始走完整的四步生成流程了。

---

## 关于"控制成本"，还可以做但目前先没做的事

- 目前的每日限额（`functions/api/generate.js` 里的 `DAILY_LIMIT`）是全站共享的一个数字，不是按人算的，先用着看效果，之后可以改成按口令分别计数
- 访问口令目前是"错了就在生成时报错"，还没有做成一进网站就校验对错的形式，如果同事反馈体验不好，可以再补一个校验接口
- 如果之后要换成真正的用户登录（每个人一个账号），或者要把某几章的数字接上真实财务测算而不是AI示例框架，这些都是在现在这个骨架上继续加，不用推倒重来


# 升级版（v3）更新说明

本次升级的核心：新增两个领域、章节结构按真实报告细化、支持表格。

## ⚠ 最重要：这次多了一个新文件 outlines.js

相比你现在线上的版本，这一版**多了一个文件 `outlines.js`**（放在和 index.html 同一层，也就是仓库根目录）。这个文件装的是两个领域的章节大纲。**如果只更新 index.html 而漏传 outlines.js，网站会打不开领域选择页**，务必一起上传。

## 需要更新/新增的文件清单

| 文件 | 在仓库里的位置 | 操作 |
|---|---|---|
| `index.html` | 根目录 | 替换（整个换掉） |
| `outlines.js` | 根目录 | **新增**（仓库里原来没有，需要新建） |
| `functions/api/generate.js` | functions/api/ | 保持不变（和上一版一样，无需动） |
| `functions/api/verify.js` | functions/api/ | 保持不变 |

### 在 GitHub 上怎么新增 outlines.js
1. 打开仓库根目录 → Add file → Create new file
2. 文件名填 `outlines.js`
3. 把本文件夹里的 `outlines.js` 内容整个粘贴进去
4. Commit

然后再把 `index.html` 的内容替换掉，一起提交。Cloudflare 会自动重新部署。

## 环境变量和 KV

**都不用动**，和现在完全一样（`ACCESS_CODE`、`DEEPSEEK_API_KEY`、`USAGE_KV` 绑定继续沿用）。

## 这一版新增/改进了什么

1. **多了第一步"选择领域"**：进入后先选"保障房·非居改保"或"保障房·新建"，两个领域用各自的专属章节大纲。
2. **章节结构大幅细化**：从旧版"一章一段"改成"一章下含多个子标题，逐个子标题撰写"。非居改保有 13 章约 40 个子标题，新建有 13 章约 40 个子标题，单份报告篇幅和结构颗粒度显著提升。
3. **数据类子章节自动生成表格**：投资估算、财务评价等涉及数字的子标题，会生成结构完整的表格框架，数字统一填"待填"，并醒目标注需替换为真实测算——既贴合真实报告的表格形态，又守住"不编造数字"的边界。
4. **Word 导出支持表格和多级标题**：导出的 .doc 里，一级章节、子标题、表格都会保留层级和边框。

## 关于篇幅

单份报告子标题多、每节 500-800 字，生成时会**逐个子标题调用 AI**，所以一份完整报告大约要调用 40 次左右、耗时几分钟，也会相应消耗更多 DeepSeek 额度。如果只想快速看某几章，可以在"章节范围"那一步取消勾选不需要的整章。

## 关于"待填"数字（一定要跟领导讲清楚的边界）

这一版能把报告的**结构、行文、表格形态**做得非常接近真实报告，但表格里的具体数字仍然是"待填"占位——因为投资估算、经营收入、回报率这类数字必须来自真实测算，AI 编出来的数字看着专业却经不起审查。所以这套工具的定位是"把 80% 的结构和文字骨架自动搭好，剩下的真实数据由专业人员填入并复核签发"，而不是"一键出终版报告"。这条边界守住了，批量生产才有意义。
